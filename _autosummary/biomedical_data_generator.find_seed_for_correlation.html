biomedical_data_generator.find_seed_for_correlation

biomedical_data_generator.find_seed_for_correlation(n_samples, size, rho_target, structure='equicorrelated', metric='mean_offdiag', threshold=0.65, op='>=', tol=0.02, start_seed=0, max_tries=500)[source]

Try seeds until the empirical correlation satisfies the rule.

Try seeds starting from start_seed until one of the following is satisfied:

  • |mean_offdiag - rho_target| <= tol (if tol is not None), else

  • (metric op threshold) with metric in {“mean_offdiag”, “min_offdiag”} and op in {“>=”, “<=”}.

Parameters:

  • n_samples (int) – Number of samples (rows).

  • size (int) – Number of features (columns).

  • rho_target (float) – Target correlation between features.

  • structure (Literal['equicorrelated', 'toeplitz']) – “equicorrelated” or “toeplitz”.

  • metric (Literal['mean_offdiag', 'min_offdiag']) – Empirical metric to use for acceptance.

  • threshold (float) – Threshold for the metric (if tol is None).

  • op (Literal['>=', '<=']) – Operator for threshold comparison (“<=” or “>=”).

  • tol (float | None) – Optional tolerance around rho_target for acceptance.

  • start_seed (int) – First seed to try.

  • max_tries (int) – Maximum number of seeds to try before giving up.

Returns:

  • seed (int): The first seed that satisfied the condition.

  • meta (dict): Metadata as returned by generate_correlated_cluster.

Return type:

tuple

Raises:

  • RuntimeError –

  • ValueError –
